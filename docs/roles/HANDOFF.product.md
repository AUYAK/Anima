## Что такое Anima
Мобильное приложение для ухода за питомцами.
**Целевая аудитория:** владельцы домашних животных (любые виды и породы).
**Платформы:** iOS и Android.
**Монетизация:** отсутствует, приложение бесплатное.

## Продуктовое видение
Единый цифровой дневник питомца. Всё важное в одном месте:
- Когда была прививка, стрижка, визит к врачу
- Как питомец рос (вес, рост, фото)
- Напоминания о предстоящих событиях
- Курсы обучения и дрессировки

## Модель данных (упрощённо, без технических деталей)
- Один пользователь — много питомцев
- У питомца: имя, вид, порода, дата рождения, фото
- События: тип (прививка / груминг / день рождения / визит к врачу), дата, заметки
- Фотогалерея привязана к питомцу
- Замеры: вес, рост, дата
- Курсы: название, шаги, прогресс

## Как работать с требованиями

### Создание фичи
1. Проверь `docs/product/features/` — нет ли уже такого файла
2. Создай файл `docs/product/features/NNN-name.md`
3. Используй шаблон из `docs/DOR.md`
4. Статус в заголовке: Draft → Ready → (дев берёт в работу) → Done

### Чеклист перед передачей в разработку (DOR)
- [ ] Цель фичи в одном предложении
- [ ] User stories написаны
- [ ] Acceptance criteria — каждый пункт проверяемый
- [ ] Список экранов и навигация описаны
- [ ] Пустые состояния и ошибки описаны
- [ ] Явно написано что НЕ входит в фичу

### Правила
- Описывай ЧТО пользователь делает, не КАК это реализовано
- Не упоминай технологии (Flutter, Supabase, API и т.д.)
- Если что-то непонятно технически — пиши заметку "Dev: уточнить ..."
- Не редактируй файлы вне `docs/product/`
